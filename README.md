# Splunk Incident Analysis: Brute Force and Security Event Investigation

## Overview
This project focused on using Splunk to investigate security events in a lab environment, including brute-force activity and event correlation across multiple systems.

## Objective
- Review log data from multiple agents
- Search and filter events in Splunk
- Identify brute-force indicators and suspicious activity
- Document incident-response style findings

## Tools Used
- Splunk
- Virtualized lab environment
- Linux systems
- Security logs

## Environment
Logs were collected from multiple lab agents and reviewed in Splunk to simulate security monitoring and incident investigation workflows.

## Steps Performed
1. Connected and reviewed logs from multiple systems
2. Ran Splunk searches to identify relevant events
3. Investigated suspicious authentication behavior
4. Reviewed fields and event details for signs of brute-force attempts
5. Documented the findings and incident context

## Key Findings
- Splunk can be used to centralize logs from multiple systems for investigation
- Brute-force activity can be identified through repeated login-related events
- Search filters and event fields help narrow down suspicious behavior quickly
- Proper documentation is important for incident response and escalation

## Security Impact
This project reinforced how SIEM tools support incident triage and investigation by making it easier to identify patterns across multiple log sources.

- SIEM exposure
- Log analysis
- Incident investigation
